# 唱片管理库 - 网页版

一个功能完整的唱片（音乐专辑）管理应用，针对手机和平板设备优化，支持条形码扫描和完整的收藏管理。

## 📱 功能特性

### 核心功能

#### 1. 唱片管理
- ✅ **唱片列表** - 网格视图展示所有唱片，显示封面、名称、歌手、评分等
- ✅ **搜索功能** - 按名称、歌手、分类快速搜索唱片
- ✅ **多条件筛选** - 支持按分类快速筛选
- ✅ **灵活排序** - 按添加时间、名称、歌手、评分排序
- ✅ **唱片详情** - 完整信息展示（曲目、购买渠道、备注、评分等）
- ✅ **编辑修改** - 编辑已添加的唱片信息
- ✅ **删除功能** - 单个删除或批量删除，带二次确认保护

#### 2. 唱片添加
- ✅ **手动添加** - 完整的表单输入
  - 唱片名称、歌手（必填）
  - 发行日期、分类、评分
  - 格式选择（黑胶、光盘、磁带、数字）
  - 购买渠道、曲目列表
  - 个人备注和封面上传

- ✅ **条形码扫描** - 核心功能
  - 调用设备摄像头实时扫描
  - 支持条形码、二维码、ISBN识别
  - 自动从公开数据库获取信息
  - 自动填充表单，支持手动修正

#### 3. 分类管理
- ✅ **预设分类** - 流行、摇滚、爵士、古典、嘻哈、电子、民谣
- ✅ **自定义分类** - 新增自己的分类方式
- ✅ **批量关键** - 支持批量修改唱片分类
- ✅ **分类统计** - 显示每个分类下的唱片数量

#### 4. 用户体验
- ✅ **响应式设计** - 完美适配手机、平板、桌面
- ✅ **离线支持** - 数据保存到本地，无需网络
- ✅ **批量操作** - 支持批量选择、删除、分类
- ✅ **流畅动画** - 平滑的页面转换和交互反馈

## 🚀 快速开始

### 安装
1. 将所有文件放在同一文件夹
   - `index.html` - 主页面
   - `styles.css` - 样式表
   - `app.js` - 应用逻辑

2. 在浏览器中打开 `index.html`

### 系统要求
- 现代浏览器（Chrome、Firefox、Safari、Edge）
- 支持 `localStorage` API
- 对于扫码功能：需要摄像头权限

## 📖 使用指南

### 添加唱片

#### 方式一：手动添加
1. 点击右上角 **➕** 按钮
2. 选择 **手动添加** 标签
3. 填写唱片信息：
   - 必填项：唱片名称、歌手
   - 可选项：发行日期、分类、评分、格式、购买渠道、曲目、备注
4. 点击 **保存唱片**

#### 方式二：扫描二维码/条形码（推荐）
1. 点击右上角 **➕** 按钮
2. 选择 **扫描二维码** 标签
3. 点击 **开启摄像头**
4. 对准唱片条形码或ISBN
5. 系统自动识别并填充信息
6. 手动修正（如需要）后保存

### 管理唱片

#### 搜索
- 点击顶部 🔍 按钮
- 输入唱片名称、歌手或分类
- 实时搜索结果

#### 筛选
- 点击筛选栏的分类标签快速筛选
- 显示该分类下的所有唱片

#### 排序
- 点击 **排序** 按钮
- 选择排序方式：
  - 最新添加
  - 最早添加
  - 名称 (A-Z)
  - 歌手 (A-Z)
  - 评分最高

#### 查看详情
- 点击唱片卡片查看完整信息
- 可以在详情页面编辑或删除

#### 编辑唱片
1. 点击唱片卡片进入详情页
2. 点击 **编辑** 按钮
3. 修改信息后点击 **保存唱片**

#### 删除唱片
- **单个删除**：在详情页点击 **删除**
- **批量删除**：长按唱片卡片进入选择模式，选中多个，点击 **删除**

### 分类管理
1. 点击底部导航 🏷️
2. 查看所有分类及其唱片数量
3. 可以添加新的自定义分类
4. 删除自定义分类（预设分类无法删除）

### 批量操作
1. 长按任意唱片卡片进入选择模式
2. 点击卡片选择多张唱片
3. 使用底部操作栏：
   - **分类** - 批量修改分类
   - **删除** - 批量删除
   - **取消** - 退出选择模式

## 💾 数据存储

所有数据保存在浏览器的 `localStorage` 中：
- `musicRecords` - 唱片数据
- `musicCategories` - 自定义分类

### 导出/备份
目前没有内置的导出功能，但可以通过浏览器开发者工具：
```javascript
// 在控制台输入以下命令导出数据
JSON.stringify(JSON.parse(localStorage.getItem('musicRecords')), null, 2)
```

### 清空数据
```javascript
localStorage.clear()
```

## 🔧 高级功能

### 条形码扫描工作原理
1. 使用 `jsQR` 库识别二维码和某些条形码格式
2. 调用 Open Library API 查询书籍信息（可扩展到其他API）
3. 自动填充识别到的字段

### 自定义分类
除了预设的8个分类外，可以创建：
- 按风格：摇滚、民谣、电子等
- 按年代：70年代、80年代等
- 按地区：日本、英国等
- 按收藏状态：在听、待听、最爱等

## 🎯 使用建议

1. **首次使用**：建议先手动添加几张唱片，熟悉界面
2. **扫码功能**：需要唱片的条形码或ISBN，通常在背面
3. **定期备份**：如果重要，定期拷贝 localStorage 数据
4. **分类设计**：提前规划分类方式，便于管理

## 📝 键盘快捷键

目前没有键盘快捷键，所有操作都通过触摸/点击完成。

## 🐛 已知限制

1. 扫码功能需要有效的网络连接（调用API）
2. 大量唱片（1000+）时可能影响性能
3. 不支持导入/导出功能（可手动实现）
4. 不支持云同步（仅本地存储）
