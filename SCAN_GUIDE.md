# 扫码功能更新说明

## 📸 两种扫码方式

改进后的扫码模块现在支持两种方式来添加唱片：

### 1. 📷 摄像头扫描（原有功能）
- **使用场景**：当你手持唱片，想要实时扫描条形码时
- **操作步骤**：
  1. 点击 ➕ 按钮添加唱片
  2. 选择 **扫描二维码** 标签
  3. 点击 **📷 摄像头** 子标签
  4. 点击 **开启摄像头** 按钮
  5. 对准唱片条形码或ISBN，保持光线充足
  6. 系统自动识别并填充信息

**优点**：
- 实时扫描，快速高效
- 适合手持唱片扫描
- 无需预先拍照

---

### 2. 📁 图片上传扫描（新增功能）
- **使用场景**：
  - 唱片不在身边，但有条形码的照片
  - 想要先拍照再处理
  - 网络不稳定，想离线扫描

- **操作步骤**：
  1. 点击 ➕ 按钮添加唱片
  2. 选择 **扫描二维码** 标签
  3. 点击 **📁 上传图片** 子标签
  4. **两种上传方式**：
     - **点击上传区域**：选择本地图片文件
     - **拖放上传**：直接拖动图片到上传区域
  5. 选择包含条形码的图片
  6. 点击 **扫描图片** 按钮
  7. 系统识别条形码并填充信息

**优点**：
- 灵活方便，可以扫描已有的照片
- 支持拖放操作（更便捷）
- 可以多次扫描同一张图片
- 不需要实时摄像头

## 🎯 使用提示

### 最佳实践
1. **确保光线充足** - 无论是摄像头还是图片，都需要清晰的条形码
2. **对准条形码** - 条形码应该占据视图的中央位置
3. **静止不动** - 使用摄像头时保持稳定
4. **支持多种格式**：
   - ✅ 条形码（Barcode）
   - ✅ 二维码（QR Code）
   - ✅ ISBN 条码

### 常见问题

**Q: 识别失败怎么办？**
A: 可以手动填写信息，或者尝试：
- 调整光线
- 确保条形码完整清晰
- 尝试另一个角度

**Q: 上传什么格式的图片？**
A: 支持常见格式：JPG、PNG、GIF、WebP 等

**Q: 能识别国际条形码吗？**
A: 系统根据 Open Library API 识别，主要支持 ISBN。可以自行修改 searchRecordByBarcode 函数来集成其他 API（如 Discogs、MusicBrainz）

**Q: 拖放上传不工作？**
A: 某些浏览器可能限制拖放功能，此时直接点击选择文件即可
